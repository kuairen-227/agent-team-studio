# TypeScript コーディング規約

## JSDoc / TSDoc コメント方針

**すべての `export` シンボルに JSDoc を付与する（ライブラリ標準）。**
型・クラス・関数・定数を問わず、`export` が付くものは最低 1 行のサマリーを書く。

**書き方の原則:**
- 日本語で記述する
- 設計上の根拠・SSoT 参照（`docs/design/*.md`）を残す
- 「何をするか」より「なぜそうなのか」を優先する
- TypeScript の型が自明な情報（型名の言い換えなど）は繰り返さない

**`@param` / `@returns` の使いどころ:**
挙動・制約・エラーの起点が名前と型だけでは読み取れない時のみ使う。型が十分に語っている場合は省略する。

**ファイル冒頭のモジュール説明:**
モジュールの目的・責務・SSoT 参照を記述する。個別 export の JSDoc と役割を分離し、重複させない。
