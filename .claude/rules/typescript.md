# TypeScript コーディング規約

## JSDoc / TSDoc コメント方針

**必須（常に付与）:**
- ファイル冒頭のモジュール説明（目的・SSoT 参照）
- クラス
- 挙動・エラー・副作用が非自明な関数

**推奨（ホバー時に追加情報を提供する）:**
- ドメインオブジェクト型・discriminated union（SSoT 参照が有用）
- ルートファクトリ（HTTP メソッド/パスを明記）
- DI コントラクト（`AppDeps` 等、テスト利用の文脈が重要なもの）

**省略（名前・型・既存コメントで自明）:**
- 単純な型エイリアス（`TemplateId = string` 等）
- `as const` 配列から派生した union 型
- `*Deps` 型（フィールドが自己説明的な場合）
- 名前から自明なファクトリ関数（`createTemplatesService` 等）
- ファイルヘッダで説明済みのテストフィクスチャ

**書き方の原則:**
- 日本語で記述する
- 設計上の根拠・SSoT 参照（`docs/design/*.md`）を残す
- 「何をするか」より「なぜそうなのか」を優先する
- TypeScript の型で自明な情報は繰り返さない

**`@param` / `@returns`:**
挙動・制約・エラーの起点が名前と型だけでは読み取れない時のみ使う。
