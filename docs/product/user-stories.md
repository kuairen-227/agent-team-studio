# ユーザーストーリー（MVP）

MVP で提供するユーザーストーリーと受け入れ基準。前提は [ADR-0004 ターゲットユーザー](../adr/0004-target-users.md) と [ADR-0005 MVP スコープ](../adr/0005-mvp-scope.md)。

## 対象ユーザー

- **プライマリ**: 社内新規事業企画担当（ADR-0004）
- **セカンダリ**: エンジニア出身 PdM（ADR-0004、ドッグフーディング兼用）

## Hero Use Case

競合調査の並列深掘り（UC #1 / ADR-0005）。競合 3 社×4 観点の比較整理を 30 分以内に完遂することが MVP の成功基準。

## ストーリー一覧

優先度は MoSCoW 形式（MUST / SHOULD / COULD）。成功基準欄は [成功基準](#成功基準adr-0005-から転記)（1〜4）への寄与を示す。

| ID | タイトル | 優先度 | 関連機能（ADR-0005） | 寄与する成功基準 |
| --- | --- | --- | --- | --- |
| [US-1](#us-1-テンプレートを選んで調査を始める) | テンプレートを選んで調査を始める | MUST | テンプレート選択画面 | 1, 4 |
| [US-2](#us-2-調査パラメータを入力して実行する) | 調査パラメータを入力して実行する | MUST | テンプレート選択画面 / 実行基盤 | 1, 4 |
| [US-3](#us-3-エージェントの進捗をリアルタイムで見る) | エージェントの進捗をリアルタイムで見る | MUST | リアルタイム進捗表示 | 1, 4 |
| [US-4](#us-4-統合結果を閲覧しエクスポートする) | 統合結果を閲覧しエクスポートする | MUST | 結果表示画面 / 結果エクスポート | 1, 2, 3 |
| [US-5](#us-5-過去の実行履歴を振り返る) | 過去の実行履歴を振り返る | SHOULD | 実行履歴の保存 | 4 |

**優先度の根拠**:

- US-1〜US-4: JTBD「30 分以内に競合比較を完遂」の一連の動線に不可欠。いずれか欠けると Hero UC が成立しない
- US-5: ADR-0005 の MVP 機能一覧に含まれるが、単発 JTBD の完遂には直接寄与しない。ドッグフーディングと再利用価値の観点で MVP 内に残すが、スコープ調整時の最初の削減候補

---

### US-1. テンプレートを選んで調査を始める

> **As a** プライマリユーザー（新規事業企画担当）、
> **I want to** 事前定義された競合調査テンプレートを選択できる、
> **so that** プロンプトを自分で設計せずに分析を開始できる。

**受け入れ基準**:

- [ ] テンプレート一覧画面で、MVP シードテンプレート（競合調査）が表示される
- [ ] テンプレートごとに名前・説明・エージェント構成（ロール一覧）が閲覧できる
- [ ] テンプレートを選択すると、入力フォーム画面に遷移する

**備考**:

- MVP では 1 テンプレート（競合調査）のみ。ユーザーによる作成・編集は v2（ADR-0005 スコープ外）
- 一覧は将来の複数テンプレートを見越した UI にするが、MVP 段階では 1 件表示で良い

---

### US-2. 調査パラメータを入力して実行する

> **As a** プライマリユーザー、
> **I want to** 調査対象（競合企業名・観点・参考情報）をフォームで入力して実行を開始する、
> **so that** 自分のテーマで分析できる。

**受け入れ基準**:

- [ ] 選択したテンプレートのパラメータスキーマに沿って入力フォームが生成される
- [ ] 必須項目が未入力の場合、実行ボタンが押下できない（もしくはエラー表示）
- [ ] 参考情報（ユーザーが手動で貼り付けたテキスト）を任意で添付できる
- [ ] 実行ボタン押下で新規実行が作成され、進捗画面に遷移する
- [ ] 実行開始に失敗した場合（LLM API 障害・バリデーションエラー等）、原因が判別できるメッセージが表示される

**備考**:

- 入力項目: 競合企業名（複数）、観点リスト（戦略 / 製品 / 投資 / パートナーシップ等）、参考情報（任意テキスト）
- 参考情報は LLM に渡す調査コンテキストとして利用。Web 取得は行わず、ユーザーが手で貼り付けたテキストのみを受け付ける（ADR-0005 の外部検索 Non-goal と整合）

---

### US-3. エージェントの進捗をリアルタイムで見る

> **As a** プライマリユーザー、
> **I want to** 各エージェントの状態と出力をリアルタイムで確認できる、
> **so that** 待ち時間中も進捗が把握でき、途中経過から洞察を得られる。

**受け入れ基準**:

- [ ] 進捗画面で、各エージェント（観点別調査役・統合役）のステータスが「待機 / 実行中 / 完了 / 失敗」で表示される
- [ ] 実行中のエージェントの出力がストリーミング表示される（トークン単位もしくは短いチャンク単位、体感でほぼリアルタイム）
- [ ] エージェントが失敗した場合、失敗メッセージが表示される

**備考**:

- 表示粒度はエージェント単位のステータス＋出力（ADR-0005）。依存関係・思考過程の詳細可視化は v2
- 実行中断・再開はスコープ外（ADR-0005）
- 実行中のリロード時の状態復元は MVP スコープ外（[Non-goals](#non-goalsmvp-で意図的に含めないもの) 参照）。完了後の結果閲覧は US-4 で担保

---

### US-4. 統合結果を閲覧しエクスポートする

> **As a** プライマリユーザー、
> **I want to** 全エージェントの結果を統合したレポートを閲覧し、Markdown で持ち出せる、
> **so that** 提案資料のドラフトに着手できる。

**受け入れ基準**:

- [ ] 全エージェント完了後、統合レポートが結果画面に表示される
- [ ] 結果は観点×競合のマトリクス形式で表示される
- [ ] 結果全体を Markdown としてコピーできる
- [ ] 結果全体を Markdown ファイルとしてダウンロードできる
- [ ] 統合エージェントが失敗した場合、個別エージェントの結果は確認できる
- [ ] 全エージェントが失敗した場合、失敗した旨と原因（判別可能な範囲）が表示される
- [ ] 完了した実行の結果画面はページリロード後も閲覧できる

**備考**:

- マトリクス形式は MVP の固定フォーマット。カスタマイズは v2
- 出力品質判定（構造・網羅性）は ADR-0005 成功基準 3 に対応

---

### US-5. 過去の実行履歴を振り返る

> **As a** プライマリユーザー、
> **I want to** 過去の実行履歴を一覧・閲覧できる、
> **so that** 以前の分析結果を再利用・比較できる。

**受け入れ基準**:

- [ ] 履歴一覧画面で、過去実行のテンプレート名・実行日時・ステータスが閲覧できる
- [ ] 新しい順にソートされる
- [ ] 一覧から任意の実行を選択すると、結果画面（US-4 と同じ画面）に遷移する
- [ ] 履歴が 0 件のときは「まだ実行がありません」等の空状態メッセージが表示される

**備考**:

- 検索・絞り込みは MVP スコープ外（v2 以降）
- シングルユーザー前提のためアクセス制御は不要（ADR-0005）

---

## Non-goals（MVP で意図的に含めないもの）

ADR-0005「スコープ外」の具体化。

- ユーザー認証 / マルチユーザー / アクセス制御
- テンプレートのユーザー作成・編集 UI（シードテンプレートのみ）
- UC #2（フィードバック分析）、UC #3（論点整理）
- AI によるチーム構成提案
- Web 検索・外部データソース連携
- エージェントの実行中断・再開・リトライ
- 実行中のページリロード時の状態復元（完了後の結果閲覧は US-4 で担保）
- 実行履歴の検索・タグ・フィルタ
- 結果フォーマットのカスタマイズ
- 結果共有（URL 共有・コメント等）
- モバイル最適化 UI

## 成功基準（ADR-0005 から転記）

1. プライマリ JTBD シナリオ（競合 3 社×4 観点）を **30 分以内**に完遂できる
2. 手動比 **50% 以上**の時間削減を観測する
3. 結果を基に提案資料ドラフトに着手できる構造・網羅性がある
4. ドッグフーディングで「手動より明確に楽」と判断できる

- [ ] **データモデル**（Template / Execution / Result の関係、状態遷移）→ `docs/design/data-model.md`（新規、別 Issue で先行着手）
