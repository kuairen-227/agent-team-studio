## 1. 目的とスコープ

ADR-0005 §MVP 成功基準 4 軸の検証（C 系 / D 系）が完了し、総合判定は **CONDITIONAL PASS**。CONDITIONAL の引き金（出典 URL 欠如・ファクトチェック未実装・待機時 UX）と dogfooding で抽出された v2 候補（V1〜V10）、および ADR-0005 §v2 以降のロードマップ新機能候補を統合して優先順位を仮置きする。

**Issue #193 受入条件「E1 が close されている」充足確認**: C 系（#198 / #200〜#203）/ D 系（#199 / #204〜#205）/ 統合サマリ（#192）はすべて close 済み。残る v1 マイルストーンの open は #195（README 健全性）/ #194（ドキュメント整合性）/ #193（本 PR）/ #116（親トラッカ）で、いずれも検証系 Issue ではない。

## 5. v0.6 着手候補（1〜3 個に絞り込み）

**v0.6 のテーマ案**: 「CONDITIONAL → PASS 化」— MVP 検証で残った課題のうち、UI / プロンプト側で完結する 2 件を主軸に置く。

### 主軸候補（A 優先度から選定）

| 順位 | 候補 | 理由 |
| - | --- | --- |
| **1** | **V3 待機時 UI 構造化** | 基準 4 CONDITIONAL の引き金「離脱衝動」を直接解消。UI 完結で依存が少なく、実装規模 M。早期に PASS 化できる |
| **2** | **V1 出典 URL 表示**（Web 検索連携（N2）は含まない範囲にスコープを絞る） | 基準 3 CONDITIONAL の引き金。v0.6 では「LLM 知識ベース由来の情報」と「ユーザー提供の参考テキスト由来の情報」のいずれにも出典・参照元を明示するプロンプト改修に留め、外部 Web 検索を伴う本格化は v0.7 以降の N2 で実施する段階的アプローチ。ペルソナ JTBD（「役員提案の裏付けを 30 分以内に固める」）の文脈では、確度ラベルと出典が対になることでユーザーが各セルを自力で検証・棄却でき、ブランド軸「判断を奪わない」とも整合する |
| **3** | **V4 入力時の透明性向上** | 軽量改修（S）で基準 4 の追加 PASS 余地を確保。V3 と合わせて基準 4 を確実に PASS 化する。完了条件は §4 V4 行の通り「調査観点 4 つ固定をフォームに表示し、何が調査されるか実行前に把握できるようにする」 |

### 別ラインで v0.6 に同居させる既起票分

- **V5 LLM エラー画面**（#214）
- **V10 `max_tokens` 設計**（#212）

これらは既起票で個別に進捗するが、v0.6 マイルストーンに紐付けて同時並行で処理する。

### v0.6 から外す主な候補（次マイルストーン以降）

- **V2 ファクトチェック / N2 Web 検索**: 価値は高いが規模が大きく、v0.6 で抱え込むとスコープ肥大。V1 を先行させて出典明示の足場を作り、v0.7 で V2 / N2 をセットで着手する案を推奨
- **V9 大規模シナリオ再検証**: V1 / V3 実装後に再計測すると「リファインメント効果」も併せて評価できるため、v0.6 末 〜 v0.7 頭で実施するのが効率的。**着地点（受入紐付け先）は次工程の判断事項として残す** — 候補は (a) v0.7 マイルストーンの受入条件に追加 / (b) #116 親トラッカの残課題リストに追記、のいずれか。v0.6 マイルストーン定義時に決定する
- **V6 / V7**: 価値はあるが基準 PASS 化との直接関係が弱い。v0.7 以降に回す

## 6. ADR 起票要否

**結論**: 本仮置きの範囲では **ADR 不要**。本メモを v0.6 マイルストーン定義時の議事録として参照する。

判断根拠:

- v0.6 主軸候補（V1 / V3 / V4）はいずれも ADR-0005 §v2 ロードマップの「Web 検索連携」「テンプレート編集」「AI チーム構成提案」とは独立した既存機能のリファインメント
- ADR-0005 §v2 ロードマップの順序付け（特に N2 Web 検索を v0.7 以降に後送りする判断）は本メモで言及するに留め、実際に v0.7 マイルストーンを立てる時点で必要なら ADR 化（例: 「v2 ロードマップ順序決定 ADR」）を検討する
- 本メモは「仮置き」であり、v0.6 マイルストーン確定時の再検討で順序が変わる余地を残す

ADR 起票が必要になるトリガー:

- v0.6 で ADR-0005 §v2 新機能（N1〜N4）のいずれかを採用する場合
- v0.6 で MVP の「一方向実行」「テンプレート起点のみ」等の基本前提を覆す変更を行う場合
