# MVP 受入確認プロトコル

- **対象マイルストーン**: v1 - MVP Implementation
- **検証対象**: ADR-0005 成功基準 1（30 分以内完走）/ 成功基準 2（手動比 50% 削減）
- **根拠 Issue**: #198 / 子 Issue #200–203

---

## 1. 固定シナリオ

再現性を確保するため、手動・MVP 両計測で同一シナリオを使用する。

### 対象企業（3 社）

| # | 企業名 | 分類 |
| - | --- | --- |
| 1 | Dify | AI アプリ開発プラットフォーム |
| 2 | n8n | ワークフロー自動化（セルフホスト対応） |
| 3 | Zapier | ワークフロー自動化（SaaS） |

### 調査観点（4 種）

| # | 観点 | 調査内容の例 |
| - | --- | --- |
| 1 | 戦略 | 事業方向性・ミッション・最近の方針変更 |
| 2 | 製品 | 主要機能・UX・直近のリリース・差別化 |
| 3 | 投資 | 資金調達状況・ラウンド・財務健全性 |
| 4 | パートナーシップ | 提携・連携サービス・エコシステム拡大 |

### 参考情報

- 各社の最新プレスリリース・ブログ記事（直近 3 ヶ月分）からテキストを抜粋して参考情報欄に入力する
- URL ではなくテキストを貼り付ける（MVP 仕様に準拠。Web 取得は行わない）
- 両計測（手動 / MVP）で同一のテキストを使用する
- 参考情報の目安: 各社 300〜500 字程度
- **収集タイミング**: 参考情報テキストは実施者自身が C2 タイマー開始前に収集する。第三者・別 LLM・Claude Code 等に事前収集を委ねた場合は手動計測値が過小評価となるため、計測ログに収集経緯と推定収集時間を明記し、§4 成功基準 2 の「実質手動時間」算定に反映する

## 2. 手動ベースライン計測手順（C2 #201）

同一シナリオを LLM 単発チャットで実施する。

### 実施条件（手動）

- アプリ（Agent Team Studio）は使用しない
- 実施者: 1 名（n=1 運用のため C2 / C3 ともに同一）
- 使用 LLM: Claude.ai（モデルは実施前に固定し、計測ログに明記する）
- LLM チャット 1 セッションのみ使用（追加セッション不可）
- 参考情報テキストはプロンプトに含めてよい

### 計測手順（手動）

1. タイマーを開始する
2. LLM チャットに以下の内容を入力して送信する：
   - 調査対象（Dify / n8n / Zapier）
   - 調査観点（戦略 / 製品 / 投資 / パートナーシップ）
   - 参考情報テキスト
   - 出力形式の指示（観点×企業のマトリクス形式）
3. LLM の応答を確認し、不足があれば追加プロンプトで補完する
4. 結果が「提案資料のドラフトに着手できるレベル」に達したと判断した時点でタイマーを止める（目安：観点×企業のマトリクス全セルに内容が記載されていること）
5. 計測ログに記録する

### 記録項目（手動）

- 所要時間（分：秒）
- 追加プロンプト回数（手戻り回数）
- 出力の主観的網羅性（5 段階）
- 体感メモ（途中で感じた不安・迷い等）

## 3. MVP 実行計測手順（C3 #202）

同一シナリオを Agent Team Studio（MVP）で実施する。

### 実施条件（MVP）

- C2（手動ベースライン計測）が完了していること
- PR #190 (US-5) がマージ済みで全機能が利用可能な状態
- 実施者: 1 名（n=1 運用のため C2 / C3 ともに同一）
- アプリのバージョン（コミット SHA）を計測ログに明記する

### 計測手順（MVP）

1. タイマーを開始する（アプリ起動前）
2. テンプレート選択画面で「競合調査テンプレート」を選択する
3. 入力フォームに以下を入力する：
   - 競合企業名: Dify / n8n / Zapier
   - 調査観点: 戦略 / 製品 / 投資 / パートナーシップ
   - 参考情報: 固定シナリオと同一テキスト
4. 実行を開始する
5. 結果画面のマトリクス全セルに内容が表示されていることを確認した時点でタイマーを止める。エラー・欠損セルがある場合は完走不能として記録する
6. 計測ログに記録する

### 記録項目（MVP）

- 所要時間（分：秒）
- 体感メモ（待ち時間中の不安・離脱衝動・操作迷い）
- エラー発生の有無と内容

## 4. 合否判定基準

> **注**: 学習プロジェクトの実態（1 名運用）に合わせ n=1・単発計測とする。統計的有意性は主張しないが、定量計測値・定性観察値を残すことで合否判定の根拠とする。

### 成功基準 1: 30 分以内完走

| 判定 | 条件 |
| --- | --- |
| PASS | MVP 計測値が 30 分 00 秒以内 |
| CONDITIONAL PASS | MVP 計測値が 30 分 01 秒〜45 分。改善余地ありと記録し v2 で対応 |
| FAIL | MVP 計測値が 45 分超 or 完走できない |

### 成功基準 2: 手動比 50% 削減

削減率 = (実質手動時間 − MVP 計測値) / 実質手動時間 × 100

ここで **実質手動時間** は、C2 計測値に参考情報の収集時間を加算した値とする。これは ADR-0005 が想定する「手動 3 時間→MVP 1.5 時間以内」が情報収集を含む業務全体の時間を基準にしているためである。MVP（C3）は調査エージェントが情報収集フェーズを含めて実行するため、両者を同じ前提で比較するには C2 側に収集時間を加算する必要がある。

- **C2 で実施者自身が収集した場合**: 計測ログに記録した「収集時間（実測）」を C2 計測値に加算する
- **C2 で事前収集を委ねた場合（§1 参考情報「収集タイミング」例外）**: 計測ログに記録した「推定収集時間」を C2 計測値に加算する。推定根拠（参考情報の文字数 / 情報源数 / 一次情報の探索難度等）を §6 集計欄に明記する

| 判定 | 条件 |
| --- | --- |
| PASS | 削減率 50% 以上 |
| CONDITIONAL PASS | 削減率 30% 以上 50% 未満。原因切り分けと v2 改善方針を記録 |
| FAIL | 削減率 30% 未満（50% 目標に対し、v2 改善でリカバリできない水準と判断するライン） |

### 定性基準（ADR-0005 成功基準 3・4）のスコープ

成功基準 3（出力品質）・4（体感）は本プロトコルの定量計測スコープ外とする。C3 の体感メモと出力結果をもとに、C4 完了時に実施者が以下のチェックリストで判定する。

#### 成功基準 3（出力品質）チェックリスト

- [ ] §1 で定義した観点 × 競合の全セル（4 観点 × 3 社 = 12 セル）に内容が記載されているか（§3 手順 5 で完走確認済みであることを前提とする）
- [ ] 出力が役員提案資料のドラフトとして使えるレベルか（Yes / No + 理由）
- [ ] 明らかな事実誤りが含まれていないか（手動ベースライン出力との突き合わせで確認）

#### 成功基準 4（体感）チェックリスト

- [ ] 待ち時間中に離脱したくなる衝動があったか（Yes / No + タイミング）
- [ ] 操作迷いが発生した箇所があったか（Yes / No + 該当画面）
- [ ] 同じシナリオを次回も MVP で実行したいと感じるか（Yes / No + 理由）

n=1 のため厳密な閾値を設けず、上記 3 項目の回答を踏まえて実施者が PASS / FAIL を総合判断し、判定理由を §7 に記録する。

> **参照元**: §7 記入時は c3-log.md の出力結果セクション（成果物の品質確認用）および体感メモセクション（成功基準 4 の判断材料）を参照する。§6 の C3 集計表には品質評価欄を設けず、評価の起点は c3-log.md に集約する。

### FAIL / CONDITIONAL 時の原因切り分け軸

- **モデル**: LLM の応答品質・速度が手動と比較して劣っているか
- **プロンプト**: エージェントのプロンプト設計に問題があるか
- **並列度**: 並列実行が時間短縮に寄与しているか（直列と比較）
- **UX**: 入力手順・待機体験・結果表示に改善余地があるか

### 集計

- C2 計測値: 0.5 分（0分30秒）
- 参考情報収集時間（推定）: 10 分
- **実質手動時間**: 10.5 分（C2 計測値 + 推定収集時間）
- MVP 計測値: 1.42 分（1分25秒、= 85 ÷ 60）
- 削減率: ≈ 86.5%（= (10.5 − 1.42) / 10.5 × 100）

> **⚠️ 計測条件の注意**: C2 手動計測では参考情報テキスト（各社 357〜393 字、合計 約 1,111 字）を Claude Code が事前収集した。本来の手動作業には情報収集時間が含まれるため、C2 の 0分30秒は過小評価である。削減率は §4 成功基準 2 の規定に従い「実質手動時間」（C2 計測値 + 推定収集時間）を基に算定した。推定値の妥当性は C4 合否判定で評価する。
