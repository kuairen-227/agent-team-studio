# 基準3 再計測プロトコル（出典面 / N2 Web 検索連携）

- **対象マイルストーン**: v2.0 - Feature Expansion（軸1・基準3 決着）
- **検証対象**: ADR-0005 成功基準 3 のうち**出典面**（調査結果の各観点セルに実在する出典 URL が付与されるか）
- **位置づけ**: [ADR-0044](../../adr/0044-v2-scope-definition.md) §3 が #323（N2）の受入条件に課す「出典面の計測シナリオと合否閾値の定義」を満たす。**判定の実施は #326**（本ドキュメントは計測の枠組み・閾値のみを定める）
- **根拠 Issue**: #323（N2 実装）/ #326（基準3 再計測の判定）/ 親トラッカ #320
- **前提**: [validation-summary §4 V1](../v1-mvp/validation-summary.md)（出典 URL 欠如が基準3 CONDITIONAL の引き金）/ [acceptance-protocol §4 基準3](../v1-mvp/acceptance-protocol.md)（MVP の定性判定）/ [ADR-0045](../../adr/0045-web-search-api-selection.md)（Tavily 採用・捏造禁止フォールバック方針）

> **注**: 基準3 PASS には本プロトコル（出典面 = N2）に加え、ファクトチェック面（V2 / #324）の充足が要る（[ADR-0044](../../adr/0044-v2-scope-definition.md) 軸1）。本ドキュメントは出典面のみを扱い、軸1 完了の合否は両面が揃った時点で #320 が判定する。

---

## 1. 固定シナリオ

再現性のため [acceptance-protocol §1](../v1-mvp/acceptance-protocol.md) と同一シナリオを用いる。

- **対象企業（3 社）**: Dify / n8n / Zapier
- **調査観点（4 種）**: 戦略 / 製品 / 投資 / パートナーシップ
- **セル数**: 4 観点 × 3 社 = **12 セル**
- **参考情報**: acceptance-protocol §1 と同一テキストを入力する（参考テキスト由来の出典との切り分けのため、内容は据え置く）
- **実行環境**: `TAVILY_API_KEY` を設定し、Web 検索が有効な状態で MVP（アプリ）を実行する。アプリの SHA を計測ログに記録する

---

## 2. 計測手順（#326 で実施）

1. 上記固定シナリオでアプリを実行し、完走（全セル表示）を確認する
2. 結果の各セル（および全体所見）の `sources` を抽出する
3. セルごとに以下を記録する:
   - 出典の `origin`（`web_search` / `knowledge_base` / `reference` / `estimated` / `unknown`）
   - `web_search` の場合の URL
4. `web_search` URL について到達性・関連性を検証する（§3 指標 B）
5. §4 の閾値に照らして PASS / CONDITIONAL / FAIL を判定し、§5 様式で記録する

---

## 3. 計測指標

| 指標 | 定義 | 算出 |
| - | --- | --- |
| **A. 出典付与率** | 実在の Web 由来出典（`origin: web_search` で URL あり）を 1 件以上持つセルの割合 | web_search 出典ありセル数 ÷ 12 |
| **B. 実在 URL 検証通過率** | 付与された `web_search` URL のうち、到達可能（HTTP 200 相当）かつ当該観点に関連する URL の割合 | 検証通過 URL 数 ÷ 付与 URL 総数 |
| **C. 縮退健全性** | 検索結果が得られなかったセルが、捏造 URL を出さず `knowledge_base` / `estimated` / `unknown` で明示されているか（出典取得不可の明示） | 捏造 URL を持つセルが 0 件か（boolean） |

> 指標 B の「関連性」は、URL 先が当該企業×観点の主張を裏付けているかを実施者が目視で判定する（n=1 ドッグフーディングの限界は §6 に従う）。

---

## 4. 合否閾値

基準3 の主旨は**検証可能性**であり、偽の出典は基準3 を満たさない（[ADR-0045](../../adr/0045-web-search-api-selection.md) Decision 5）。したがって指標 C（捏造ゼロ）を最優先のハード条件とする。

| 判定 | 条件（すべて満たす） |
| - | --- |
| **PASS** | A（出典付与率）≥ 75%（12 セル中 9 セル以上）/ B（実在 URL 検証通過率）= 100% / C（縮退健全性）= 捏造 0 件 |
| **CONDITIONAL PASS** | A が 50%〜75% 未満 / B = 100% / C = 捏造 0 件。出典付与の網羅は道半ばだが、付与された出典はすべて実在し捏造が無い状態。原因（無料枠・検索ヒット率・プロンプト）を切り分け v2.x で改善 |
| **FAIL** | A < 50%、または B < 100%（不達・無関連 URL が混入）、または C で捏造 URL が 1 件でも検出された場合 |

### 閾値設定の根拠

- **A を全セル必須にしない理由**: Tavily 無料枠（1,000 credits/月・繰越なし）と検索ヒット率の制約下で、全 12 セルに実 URL を強制すると無料枠運用方針（ADR-0045）と緊張する。過半（75%）に実出典が付き、残りが健全に縮退していれば、MVP の「出典 URL 欠如」課題（validation-summary §4 V1）は実質的に解消される
- **B を 100% に固定する理由**: 1 件でも不達・無関連 URL があれば「ユーザーが各セルを自力検証できる」という受入条件を破る。出典の質はトレードオフ不可
- **C をハード条件にする理由**: 捏造 URL は基準3 の主旨（検証可能性）を直接否定する。縮退時に偽 URL を出さないことは N2 の設計中核（フォールバックは `knowledge_base` 由来へ・URL 捏造禁止）

## 6. 限界

- **n=1・単発計測**: [validation-summary §5](../v1-mvp/validation-summary.md) と同じく統計的有意性は主張しない。判定根拠（セル別出典・URL 検証結果）を証跡として残し、第三者が追検証できる形で #326 / #320 に記録する
- **検索結果の非決定性**: Web 検索結果は時点依存で、再実行で URL が変わりうる。計測時点の結果を記録し、再現性は「手順とシナリオの固定」で担保する（個別 URL の固定は求めない）
- **出典面のみ**: ファクトチェック面（V2 / #324）は対象外。基準3 の最終 PASS は両面の充足を #320 が統合判定する
