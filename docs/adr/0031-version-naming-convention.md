# 0031. プロダクトのバージョン命名規約

## Status

accepted

- 作成日: 2026-05-24
- 関連: ADR-0005（MVP スコープと v2 以降のロードマップ）, ADR-0025（v0.5 表記を使用）, Issue #193（v0.6 表記を使用 / CLOSED）

## Context

プロダクトのバージョン表記がプロジェクト内で揺れている。

| 出典 | 表記 | 意味するもの |
| --- | --- | --- |
| GH マイルストーン | `v1 - MVP Implementation` | MVP の実装フェーズ |
| ADR-0025 §Context | 「MVP 実装フェーズ（v0.5）」 | MVP の実装フェーズ |
| ADR-0005 §v2 以降のロードマップ概観 | 「v2 / v3」 | MVP 完了後の機能拡張フェーズ |
| Issue #193 本文 | 「個別 Issue 化は v0.6 以降で行う」 | MVP 完了直後の改善フェーズ |

3 つの文書で「MVP」のバージョン番号が **v0.5 / v1 / (暗黙的に 1.0 未満)** として揺れており、さらに「MVP 完了直後の改善フェーズ」を指す呼称（v0.6 / v1.x / v2 のいずれか）が未定。

MVP マイルストーン完了判定（#116）と次フェーズの Issue 起票を進める段階で、命名規約の明文化が必要になった。本 ADR で単一の規約を確立する。

## Considered Alternatives

| # | 選択肢 | 判定 |
| - | --- | --- |
| A | **v1 = MVP に揃える**。v1.0 を MVP、v1.x を MVP 改善、v2.0 を機能拡張、v3.0 を UC#3 等 | **採用** — GH マイルストーン現状と ADR-0005 ロードマップ表に整合。修正対象は ADR-0025 / ADR-0030 / v2-priority-memo.md の 3 ファイル |
| B | v0.x = pre-1.0 に揃える。v0.5 を MVP、v0.6 を MVP 改善、v1.0 を初の正式リリース | 却下 — GH マイルストーン名と ADR-0005 ロードマップ表の両方を改名する必要があり、影響範囲が広い |
| C | ハイブリッド（v1 = MVP は維持、ADR-0005 の v2 / v3 も温存、用語整理は最小限） | 却下 — 案 A と本質的に同じだが、命名規約として体系的に書かれず合意が暗黙的になる |

SemVer の「最初の安定リリース = 1.0」という一般慣行とは案 A が最も近い。学習プロジェクトとして「初リリース = MVP」と位置づけることが直感的であり、既存資産（GH マイルストーン / ADR-0005 表）との整合性も最大なため、案 A を採用する。

## Decision

**プロダクトバージョンは v1 = MVP に統一する。**

### バージョン定義

| バージョン | 内容 |
| --- | --- |
| v0.x | 使用しない（既存記述で残るが、新規には使わない） |
| v1.0 | MVP（現 GH マイルストーン `v1 - MVP Implementation` に対応） |
| v1.x（v1.1, v1.2, ...） | MVP 改善・リファインメント（dogfooding / acceptance 由来の V1〜V10 等） |
| v2.0 | 機能拡張（ADR-0005 §v2 ロードマップ: AI チーム構成提案 / Web 検索連携 / UC #2-3 追加 / テンプレート作成・編集） |
| v3.0 | ADR-0005 §v3 ロードマップ: UC #3 追加 / エージェント中断・再開 / フィードバックループ |

### 適用範囲

- **新規**: ADR / Issue / マイルストーン / ドキュメントは本規約に従う
- **既存修正**（本 ADR と同 PR で実施）:
  - **ADR-0025**: 「MVP 実装フェーズ（v0.5）」→「MVP 実装フェーズ（v1.0）」
  - **ADR-0030**: Consequences 内「v0.5 完了タスク整理など」→「v1.0 MVP 完了タスク整理など」
  - **`docs/milestones/v1-mvp/v2-priority-memo.md`**: `v0.6` → `v1.1` / `v0.7` → `v1.2` を全置換
- **修正しない既存記述**:
  - **Issue #193**（CLOSED）: 本文の「v0.6 以降」は修正しない（CLOSED Issue の本文を後から書き換えると履歴の意図が不透明になるため）。本 ADR で「v1.1 以降と読み替える」ことを明示する
  - **GH マイルストーン `v1 - MVP Implementation`**: 維持

## Consequences

### ポジティブ

- プロジェクト全体で単一のバージョン体系が確立され、Issue / ADR / マイルストーン / ドキュメント間の参照が破綻しない
- 「MVP 完了 = v1.0 リリース」という直感的な区切りができ、フェーズ移行（v1 クローズ → v1.1 マイルストーン作成）の意思決定が明確化する
- 既存 GH マイルストーン名と ADR-0005 ロードマップ表との整合が取れる

### ネガティブ / リスク

- Issue #193（CLOSED）の「v0.6 以降」表記は本 ADR で読み替えを明示するに留めるため、#193 本文だけ読むと混乱する余地が残る
- SemVer の「production-ready の宣言として 1.0 を使う」という慣行とは合致しない。本プロジェクトは学習目的であり production リリースの概念がないため、影響は限定的と判断する

### 中立

- 本規約は学習プロジェクトの内部運用ルールであり、外部利用者向けの SemVer 約束ではない
- 将来パッケージを公開する場合は npm 等の SemVer 規約を別途検討する
