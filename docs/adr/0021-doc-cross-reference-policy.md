# 0021. ドキュメント間参照ポリシー（ハブ＆スポーク + 緩い水平参照）

## Status

accepted

- 作成日: 2026-05-02
- 関連: ADR-0013（前提・補完）, ADR-0017（前提）, ADR-0019（関連）

## Context

ADR-0013 で「ドキュメント配置ポリシー」を定めたが、配置されたドキュメント同士の「参照ルール」は未定義のままだった。AI スキル（write-design-doc / write-product-doc）が「相互参照を確認せよ」「双方向リンクが設定されているか確認せよ」と指示しているため、AI が docs を作成・更新するたびに参照リンクが増殖する。

実態として:

- `docs/design/ui-patterns.md` は 38 リンク（うち 7 ファイルを冒頭「前提」で列挙、§9 で同様の doc を「関連ドキュメント」として再列挙）
- `docs/product/brand.md` ↔ `docs/design/ui-patterns.md` は双方向リンク。`§2-1` `§2-2` `§6.1` 等のセクション精度の参照が双方に張られている
- `docs/design/data-model.md` 25 リンク、`docs/product/glossary.md` 33 リンク

この構造は次のコストを生む:

- 保守コスト: ファイル名・セクション番号の変更で複数 doc を直す必要があり、AI は古いリンクを残しがち
- 読み手の認知負荷: 単独で読めず、リンクを辿らないと読み終えられない
- 一方向の劣化: AI は新規・レビュー時に「関連リンクを追加」する傾向があり、リンクは増える方向にしか動かない

## Considered Alternatives

| # | 選択肢 | 判定 |
| --- | --- | --- |
| A | 完全独立（縦参照のみ許容） | 却下 — ADR の `superseded by`、glossary 参照、SSoT 接続（テンプレート I/O ↔ プロンプト仕様）など本質的に必要な水平参照が存在する |
| B | 双方向リンク既定の継続 | 却下 — 本ポリシー策定の動機 |
| C | ハブ＆スポーク + 緩い水平参照 | **採用** — README をハブとし、横方向は最小限の例外のみ許容する |

## Decision

### 用語

- **縦参照**: `README.md` → 配下 doc、または配下 doc → 親 README への参照
- **水平参照**: 同階層・他カテゴリ間の参照（例: design ↔ product、design 内の doc 同士）
- **基盤ドキュメント**: 他 doc から頻繁に参照される定義型 doc（`glossary.md` / `principles/` / 各 README / ADR）

### 許容パターン

1. 縦参照は自由
2. ADR 間の `superseded by` / 派生関係
3. 基盤ドキュメントへの参照
4. SSoT 接続が必要な水平参照（例: `product/templates/` のプロンプト仕様 → `design/templates/` の I/O スキーマ）
5. § アンカー精度の参照は SSoT 側から **初出 1 回のみ** 片方向で許容する。同一 doc 内での複数回参照は禁止

### 禁止・最小化パターン

1. 双方向の水平リンク（A → B かつ B → A）。SSoT 側に寄せて片方向にする
2. 同一 doc 内での「前提」セクションと「関連ドキュメント」セクションの二重列挙
3. 「関連ドキュメント」セクションでの広範な列挙。README で語れるなら README 経由にする

### SSoT 寄せの判断基準

- 概念の SSoT 側 → 参照 **される** 側
- 派生・適用側 → 参照 **する** 側
- 例: `brand.md`（ブランド軸の SSoT）→ `ui-patterns.md`（適用先）への参照は最小化。`ui-patterns.md` → `brand.md` の参照は許容

### 例外

- 同 PR で同時に生まれる新 doc では、初版に限り § レベル参照を 1 方向で許容する（実装着手前の発見支援のため）。実装後の整理で削除可
- 例外参照の削除判断は、新 doc が参照する側の機能を実装する後続 PR のレビュー観点として確認する（レビュアーが「§参照が依然必要か」を問う）

## Consequences

### ポジティブ

- 保守コストが低減する。ファイル移動・改名時に直すリンク数が減る
- 各 doc が単独で読める粒度になり、読み手の認知負荷が下がる
- AI スキルの指示が「最小化」に揃うため、参照増殖の劣化方向が止まる

### ネガティブ / リスク

- 精度ある発見可能性とのトレードオフ → 例外（同 PR 初版・SSoT 接続）で緩和
- 既存 docs の整理作業が発生（本 ADR 承認後の PR で順次対応）
- `glossary.md` 33 リンク・`principles/testing.md` 16 リンクは、それぞれ用語辞書の構造・principles 主題別分割（ADR-0019）に由来する構造的参照であり、本ポリシー対象外。必要に応じて別 Issue で再評価する

### 中立

- 各 README の「相互参照ルール」セクションは本 ADR へのリンクに集約する
- AI スキルの「双方向リンク確認」手順を「水平参照の最小化」に書き換える運用が必要
