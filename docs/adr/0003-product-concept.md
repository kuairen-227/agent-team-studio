# 0003. プロダクトコンセプト

## Status

accepted

## Context

ADR-0002 で社内新規事業という状況設定を定めた。次のステップとして、
「何を作るのか」を明確にする必要がある。プロダクト名「Agent Team Studio」
の由来を踏まえつつ、コンセプト・ビジョン・解決する課題を定義する。

背景にある仮説：

- LLM とエージェント技術の進展により、複数の AI エージェントが協働する
  「エージェントチーム」的なワークフローが実用レベルに達しつつある
- しかし現状、その恩恵はエンジニアや一部のアーリーアダプターに偏在しており、
  業務部門の多くは単発プロンプトの ChatGPT 利用に留まっている
- 社内新規事業としても、個人の学習目的としても、この「偏在の解消」が
  有意義な挑戦領域となる

## Decision

### プロダクトの性質

**非エンジニアの業務人材が、Web UI で AI エージェントチームを構築・実行
できる汎用プラットフォーム** として Agent Team Studio を定義する。

### ビジョン

#### 北極星ビジョン

> **誰もが、自分の AI エージェントチームを率いて働く世界を創る。**

専門知識がなくても、一人ひとりが AI チームと共に成果を出せる世界を当たり前にする。

#### MVP ビジョン（検証可能レベル、#9 で具体化）

MVP では「特定業務領域において、非エンジニアが AI エージェントチームを
使って業務時間を大幅に削減できる」ことを検証可能な形で示す。
具体的な領域・指標は #9 MVP スコープで確定する。

### 解決する課題

1. **AI 活用の格差** — 業務部門の多くが単発プロンプトの域を出られず、
   複雑な業務で AI の恩恵を受けられていない
2. **エージェント設計の壁** — 役割分担・プロンプト設計・ツール連携は
   専門知識を要し、非エンジニアが手を出せない
3. **ノウハウの属人化** — 個人の AI 活用スキルがチーム・組織に広がらず、
   スケールしない
4. **既存ツールの UX ミスマッチ** — 既存のエージェント構築ツールは
   開発者向け UX で、業務ユーザーには使いこなせない

### スコープ外（プロダクト全体として除外）

- 音声・画像生成機能（テキストベースの対話・成果物に集中）
- モバイル専用アプリ（Web レスポンシブ対応までとする）
- エンタープライズ機能（SSO、監査ログ、マルチテナント等）
- 本番運用・課金・商用利用（学習プロジェクトのため）

### 学習目標

本プロジェクトを通じて **LLM アプリケーション開発を一通り経験する**
ことを学習目標とする。具体的な技術領域・ツールの選定は、プロダクトの
要求に応じて後続の技術選定（#10）および各実装フェーズで判断する。

### 後続 Issue への作業仮説（確定は各 Issue にて）

- **#8 ターゲットユーザー**: AI に関心のある業務人材（企画・PdM・
  コンサル職等）を仮説として想定
- **#9 MVP スコープ**: 調査・情報整理系タスクを Hero Use Case の
  第一候補として想定。UX モデルはテンプレート起点 + AI によるチーム
  構成提案のハイブリッドを仮説
- **#10 技術スタック**: Node.js 24 / TypeScript strict / workspace
  プロトコルは ADR-0002 時点で既定

## Consequences

### ポジティブ

- 社内新規事業の文脈と「AI 活用の民主化」という明確な北極星を両立できる
- LLM アプリケーション開発の中核技術をひと通り体験できる
- 非エンジニア向けという絞り込みにより、UX 設計・抽象化の練度を
  意図的に高めることができる

### ネガティブ / リスク

- 「非エンジニア向け」は UX 難度が高く、開発者である PM・エンジニアが
  直接のターゲットでないため、ドッグフーディングが困難
  → 対策: #8 でペルソナを「AI 好きの業務人材」に半歩寄せ、開発者にも
  感覚が近い層を狙う
- 「汎用プラットフォーム」は MVP 段階では焦点が定まらないリスクがある
  → 対策: #9 で Hero Use Case を 1 つに絞り、そこから段階的に拡張
- 競合（Dify, CrewAI, Copilot Studio 等）が多く、差別化設計が必要
  → 対策: 「非エンジニア × 汎用 × チーム提案の自動化」の交差点で
  差別化を図る（#9 の UX 設計で詰める）

### 中立

- 本 ADR はコンセプトレベルの決定のみを扱い、ターゲット詳細・MVP 機能・
  技術選定は後続 Issue (#8, #9, #10) に委ねる
- 学習プロジェクトとしての位置付けは ADR-0002 を継承する
