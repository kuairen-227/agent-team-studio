# 0002. プロジェクトの前提・状況設定

## Status

accepted

- 作成日: 2026-04-17
- 関連: ADR-0001（前提）

## Context

このプロジェクトは AI 駆動開発（企画→要件定義→設計→開発→試験→改善→運用保守）を学習する目的で行う個人開発である。しかし、学習効果を高めるためには、実際の業務に近い判断・制約を体験できるリアリティある状況設定が必要である。

状況設定の候補として以下を検討した：

1. **スタートアップ初期** — 少人数（1-3人）で MVP を高速にリリース。リソースも時間も限られている
2. **個人開発者/インディー** — 1人で SaaS やツールを作ってリリース。AI で一人の限界を超える
3. **社内新規事業** — 企業内の小規模チームで新プロダクトを立ち上げる。社内承認やセキュリティ要件あり

## Decision

**社内新規事業** を仮想の状況設定として採用する。

- **チーム構成**: PM 1人 + エンジニア 1人（エンジニアが AI と協働して開発）
- **社内制約**: 標準的（クラウドは承認制、セキュリティレビューあり、ただし柔軟性あり）
- **開発スタイル**: エンジニアが Claude Code を活用し、企画から運用保守まで一気通貫で AI と協働

## Consequences

- プロダクト定義では「社内の課題解決」という文脈が使える
- 技術選定では社内承認を通る標準的な技術を優先する判断基準が生まれる
- セキュリティ・認証は「標準的な社内要件を満たす」レベルを目指す
- 1人のエンジニアが全工程をカバーするため、シンプルさと保守性を重視する
- 学習プロジェクトでありながら、実務に近い意思決定プロセスを体験できる
